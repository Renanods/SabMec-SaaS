Estrutura geral:
- Blueprint de pessoas: endpoint base `pessoas`
- Blueprint de itens: endpoint base `itens`

- Blueprint de OS: endpoint base `os`
- Templates em `cadastros/` e `comercial/`
- Base visual com `bases/base.html`, sidebar e componentes Bootstrap escuros.
- CSS já tem classes como:

  - `card-header-custom`

  - `form-control-sabmec`
  - `card-linha`
  - `btn-flutuante`

- Usar HTML/Jinja seguindo os padrões já existentes.
- Sempre usar `novalidate` nos forms quando necessário para evitar validação nativa do navegador atrapalhar abas/campos escondidos.

9. OS nova
- Rota `os_novo.py` usa `session` para guardar itens temporários até salvar OS.
- Chave de session: `"os_novo_itens"`.
- A OS só grava no banco ao clicar em `salvar_os`.
- Não pode gravar OS sem itens.
- Não pode gravar OS com total <= 0.
- Não pode adicionar item com quantidade <= 0, valor_unitario <= 0, desconto negativo ou total <= 0.
- Rotas AJAX:
  - `/os/buscar-clientes`
  - `/os/buscar-itens`
  - `/os/veiculos-por-cliente/<int:cliente_id>`
- O cliente e o item NÃO devem carregar todos em selects.
- No frontend, cliente e item são campos de texto com hidden id.
- Ao clicar na lupa ou pressionar Enter, abre um modal de consulta.
- Modal de clientes tem filtros:
  - Nome
  - Código
  - CPF/CNPJ
- Modal de itens tem filtros:
  - Nome
  - Código
- Ao selecionar cliente:
  - preenche `cliente_id`
  - preenche input com `#id - nome (documento)`
  - carrega veículos do cliente.
- Ao selecionar item:
  - preenche `item_id`
  - preenche tipo, descrição, valor unitário, quantidade=1, desconto_item=0.
- Houve bug de desconto duplicado porque desconto do item e desconto da OS tinham o mesmo `name="desconto"`.
- Corrigido usando:
  - desconto do item: `name="desconto_item"`
  - desconto OS: `name="desconto_os"`
  - acréscimo OS: `name="acrescimo_os"`
- A tela de OS usa abas:
  - Dados da OS
  - Produtos e Serviços
  - Pagamento
- Para não voltar para a primeira aba após adicionar/remover item:
  - hidden `aba_ativa`
  - `renderizar_os_novo` passa `aba_ativa`
  - botões de adicionar/remover item setam `aba_ativa='itens'`
  - botão salvar OS pode setar `aba_ativa='pagamento'`
  - classes das tabs usam `aba_ativa`.

10. Coisas importantes de erro já vistas
- Se der `Status` não possui `ordens_servico`, trocar em `OrdemServico`:
  - de `db.relationship("Status", back_populates="ordens_servico")`
  - para `db.relationship("Status")`
- Se `Status` tem coluna `situacao`, usar `status.situacao`, não `status.nome`.
- Se SQLite diz `no such table`, rodar `db.create_all()` ou migration.
- Se model mudou e banco não, recriar/migrar.
- Se campos monetários dão setinhas, renderizar com `type="text"` e `inputmode="decimal"`.
